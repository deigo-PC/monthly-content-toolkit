# SYNC.md — How to Work With This Repo

## One clone per machine

```powershell
git clone https://github.com/deigo-PC/monthly-content-toolkit.git
cd monthly-content-toolkit
```

## Before you work

```powershell
git checkout master
git pull
```

## Who edits what

| Folder | Who edits | Notes |
|---|---|---|
| `.agents/skills/` | Humans (authors) | Skill code — hand-edited, machine-read |
| `docs/` | Humans (authors) | Toolkit documentation |
| `SYNC.md` | Humans (authors) | This file — team contract |
| `Brands/*/AGENT.md` | Humans (brand managers) | Brand identity — set once, updated rarely |
| `Brands/*/CONTEXT.md` | Machine (pipeline) | Routing config — auto-updated |
| `Brands/*/BrandKit/` | Humans (designers) | Brand visual assets |
| `Brands/*/_knowledgebase/` | Humans (strategists) | Strategy docs, research |
| `Brands/*/image_gen_references/` | Humans (SOP authors) | Symlinks to `docs/SOP_reference/` |
| `Brands/*/hl_subaccount_config/` | Machine | GHL pipeline config |
| `Brands/*/Monthly_Cycles/` | **Machine-written** | All cycle content is generated |
| `Brands/*/brand_guide.html` | Machine | Generated by Claude |
| `Brands/*/marketing_brain.html` | Machine | Generated by marketing-brain skill |
| `Brands/*/masterStrategyDoc.html` | Machine | Generated by marketing-brain skill |
| `Brands/*/marketing_hub_url.txt` | Machine | Generated by marketing-hub-generator |

## Commit rules

- **Machine-written content** (`Monthly_Cycles/`) is committed at defined
  checkpoints (end of each stage), not ad hoc.
- **Human-edited files** (skills, docs, AGENT.md) are committed when changed,
  like any normal repo.
- **Never commit credentials** into any file — put them in `.env` or
  n8n/vault.
- **One commit per brand per operation** — makes reversion easy.

## Branching

- `master` is the single source of truth. The pipeline reads from `master`.
- No long-running feature branches. If experimenting, use a local-only branch
  and rebase onto `master` before the pipeline needs it.
